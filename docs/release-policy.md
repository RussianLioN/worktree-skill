# Release Policy

## Versioning

Use semantic versioning:

- `MAJOR`: incompatible core or adapter contract changes
- `MINOR`: new adapters, additive install flows, additive compatibility docs
- `PATCH`: fixes, clarifications, low-risk script improvements

## Release Notes Sections

- core behavior changes
- adapter changes
- Speckit bridge changes
- migration notes
- verification changes

## First Stable Release

`1.0.0` requires:

- portable `core/`
- Claude Code adapter
- Codex CLI adapter
- OpenCode adapter
- Speckit bridge docs
- install and verify scripts
- migration guide
- quickstart and compatibility matrix
